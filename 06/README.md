# 06 P2P, Relay 서버 방식

## P2P

- 방장을 정하는 방식
    - 방장이 나가는 경우 방이 폭파될 수도...
    - 새로운 방장을 만드는 방법이 있으나 로직이 복잡함
- 풀 커넥션 방식
    - 각 클라이언트가 관리해야하는 커넥션이 많음
    - 사용자가 나가더라도 게임이 중단되지 않는다

## Realy Server

Client - Server - Client

- 방장을 뽑는 방식과 유사하고 안정성이 높다 

## 차이점

- P2P 방식의 가장 큰 문제점
  - IP 매핑 도구 아래에 변동 아이피를 받기 때문에 어떻게 보내야 하는지 알 수 없다 
  - 그래서 홀펀칭이라는 방식을 사용한다
